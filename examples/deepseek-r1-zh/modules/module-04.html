Method Deep-Dive

核心算法：GRPO（Group Relative Preference Optimization）

GRPO 是 DeepSeek 团队自研的 RL 训练算法，是 R1 的心脏。它是对标准 PPO 的重大改进——不需要单独训练 Critic 网络，而是用组内相对奖励来估计 Advantage。

💬 GRPO 的工作流程
📋
问题

我有一道数学题 Q。我需要模型生成多个不同的答案。

🤖
Model (π_θ)

正在生成 G=8 个不同解：[S₁, S₂, ..., S₈]。大部分会错，但有些可能对！

✅
Reward Model

对每个解打分：[r₁=0, r₂=1, r₃=0, r₄=0, r₅=1, r₆=0, r₇=0, r₈=0]。两个正确！

⚖️
GRPO Loss Function

计算偏好：正确答案得正权重，错误答案得负权重。然后更新 π_θ，让下次生成更多正确答案。

GRPO 目标函数（完整公式）
L(θ) = − E_{(q,g)∼D} [ 1/G Σ_i min(r_i(θ)·A_i, clip(r_i(θ), 1−ε, 1+ε)·A_i) ]

其中 Advantage:    A_i = r_i − mean([r_1,...,r_G]) / std([r_1,...,r_G])
L(θ) 这是我们要最小化的损失函数。通过梯度下降更新 θ 来最大化期望奖励 R。
−E_{(q,g)∼D} 对训练集中的每个问题 q 和采样组 g，求期望（平均）损失。负号是因为我们在做梯度上升（最大化奖励），但写成最小化损失的形式。
1/G · Σ_i 对每个问题生成 G 个答案，计算 G 个损失的平均值。这 G 个答案构成了一个"组"（Group）。
min(r_i(θ)·A_i, clip(...)·A_i) 这是 PPO 的核心思想：clip 防止策略更新太激进。当 r_i(θ) 超出 [1-ε, 1+ε] 范围时，用 clip 后的值替代，防止策略偏离太远。ε 通常取 0.2。
A_i = r_i − mean(r)/std(r) 这是 GRPO 的关键创新！不与旧策略比较，而是在组内标准化——把每个答案的奖励减去组平均值再除以标准差。这样就不需要单独的 Critic 网络估计 V(s)，效率大幅提升！标准差标准化确保 A_i 分布在合理范围内，避免极端值主导梯度。

GRPO vs PPO：关键区别

PPO
PPO（Proximal Policy Optimization）

需要训练一个额外的 Critic 网络 V(s) 来估计状态价值，然后通过 GAE（Generalized Advantage Estimation）计算 Advantage。需要大量计算资源来同时更新 Actor 和 Critic。

GRPO
GRPO（Group Relative Preference Optimization）

用组内相对奖励替代 Critic 网络！对于每个问题，生成 G 个答案，计算每个答案相对于组平均的优势（A_i）。无需训练 Critic，训练速度提升 2-3 倍，显存占用大幅降低。

1
冷启动（Cold Start）

用长 CoT 数据微调基础模型，解决 R1-Zero 可读性差的问题。数据来源：可读性过滤后的 R1-Zero 数据 + 人工后处理。这一步是纯 RL 能成功的关键前提。

2
RL 阶段（面向推理）

GRPO + 奖励信号：
• 准确性奖励：答案正确得 +1，错误得 -1
• 格式奖励：模型必须把思考过程放在 <think>...</think> 标签内
• 长度奖励（可选）：适度鼓励更长的思考过程

3
SFT 阶段（面向通用能力）

用阶段 2 产生的高质量推理数据 + 来自 DeepSeek-V3 的数据，训练模型在非推理任务（写作、对话、摘要）上的能力。这一步防止模型只会推理而失去通用性。

4
RL 阶段（面向全部能力）

在阶段 3 基础上，用更通用的奖励信号做 RL：
• 有用性奖励：回答是否对用户有帮助
• 无害性奖励：回答是否安全、不含敏感内容

奖励信号设计

✓
准确性奖励（Outcome Reward）

二元判断：答案对错。最简单也最有效的奖励。模型学会在产生错误后"反思"并纠正。这种简单的奖励能驱动模型自发涌现复杂推理策略。

🎯
过程奖励（Process Reward）

给推理的每一步打分，而非只看最终答案。帮助模型理解"哪一步走错了"。但实现难度大（需要过程标注数据），论文主要使用准确性奖励。

📐
格式奖励（Format Reward）

强制模型把思考过程放在 <think>...</think> 标签内。确保输出可解析，也间接鼓励模型写出更完整的思考链。格式奖励不直接影响答案正确性，但能改善可读性。

模型架构

DeepSeek-V3-Base (671B) Dense Mixture-of-Experts 架构，37B 参数每次前向传播激活。前身 DeepSeek-V3 的创新：多头潜在注意力（MLA）+ 细粒度专家分割（DeepSeekMoE）
MLA (Multi-head Latent Attention) DeepSeek 自研注意力机制，通过低秩压缩大幅降低 KV cache 内存占用，推理效率提升约 30%
DeepSeekMoE 细粒度专家分割 + 共享专家隔离，在保持性能的同时将激活参数从 671B 降至 37B，大幅降低训练和推理成本
蒸馏: 7B / 8B / 32B / 70B 将 R1 的推理能力蒸馏到小模型，首次证明 7B 模型也能涌现推理能力（AIME 52.5%，从 2.3% 提升 22 倍）

理解检查

Q: GRPO 相比 PPO 的关键改进是什么？

A GRPO 使用更大的 batch size ✗ 错误。batch size 不是主要改进。
B GRPO 在组内做标准化（用组均值和标准差替代 Critic 网络），不需要单独的价值网络 ✓ 正确！这是 GRPO 的核心创新——省去了训练 Critic 的开销，效率大幅提升。
C GRPO 使用完全不同的强化学习算法框架 ✗ 错误。GRPO 仍是 on-policy PPO 类方法，只是对 advantage 估计做了创新。
