问题与动机

Problem & Motivation

大模型的推理困境

大型语言模型（LLM）在自然语言生成上表现优异，但在需要复杂推理的任务上却常常力不从心——数学证明、代码调试、科学分析。

📐
数学推理

多步代数推导、几何证明、奥林匹克级别竞赛题——LLM 常给出"看起来对但错了"的答案

🧩
代码生成与调试

复杂算法、多文件协调、边界条件处理——需要系统化思考而非模式匹配

🔬
科学分析

假设检验、实验设计、数据解读——需要领域知识与逻辑链条的深度结合

现有方法的三大瓶颈

1
需要大量标注数据

传统 RLHF 需要海量高质量的"人类反馈数据"——成本极高，且难以覆盖长思维链

2
必须先做 SFT（监督微调）

现有最强推理模型（GPT-4o、Claude）都需要先经过 SFT 才能做 RL——RL 只是辅助工具

3
推理链质量不可控

CoT prompting 可以引导思考，但模型自己无法验证推理步骤的正确性，容易"一本正经地胡说八道"

DeepSeek-R1 的核心目标

这篇论文要回答一个根本性问题：

RESEARCH QUESTION
"Can a large language model develop emergent reasoning capabilities
through pure reinforcement learning
without any supervised fine-tuning?"
pure RL 完全摒弃 SFT 阶段，直接用 RL 从零开始训练。这就像婴儿不学走路直接学跑步——理论上几乎不可能，但 DeepSeek-R1-Zero 做到了。
emergent reasoning 不是教模型"怎么推理"，而是创造条件让推理能力自己"涌现"出来。类似于 AlphaGo 不学人类棋谱，靠自我对弈达到超越人类水平。
冷启动策略 DeepSeek-R1 增加了冷启动阶段来解决 R1-Zero 的可读性问题——这是最终成功的关键一步。

论文贡献一览

DeepSeek-R1-Zero 首个纯 RL 训练的 671B 大模型，涌现出反思（Reflection）和长思维链能力
DeepSeek-R1 冷启动 + 多阶段训练 + SFT，性能与 OpenAI o1 持平

Distillation 7 / 8 / 32 将推理能力蒸馏到小模型，首次证明小模型也能涌现推理

方法详解

Method Deep-Dive

核心算法：GRPO

GRPO（Group Relative Preference Optimization）是 DeepSeek 团队自研的 RL 训练算法，是 R1 的心脏。

💬 GRPO 是如何工作的？
📋
问题

I have a math problem Q. I need the model to generate multiple different answers.

🤖
Model (π_θ)

Generating G=8 different solutions: [S_1, S_2, ..., S_8]. Most will be wrong, but some might be correct!

✅
Reward Model

Scoring each solution: [r_1=0, r_2=1, r_3=0, r_4=0, r_5=1, r_6=0, r_7=0, r_8=0]. Two correct!

⚖️
GRPO Loss Function

Computing preference: correct solutions get positive weight, wrong solutions get negative weight. Then update π_θ to produce more correct solutions next time.

GRPO OBJECTIVE
L(θ) = − E_{(q,g)∼D} [ 1/G Σ_i min(r_i(θ)·A_i, clip(r_i(θ), 1−ε, 1+ε)·A_i) ]

A_i = r_i − mean([r_1,...,r_G]) / std([r_1,...,r_G])
−E_{(q,g)∼D} 对训练集中的每个问题 q 和采样组 g，求期望（平均）损失。这里的"损失"前面有负号，因为我们是在最小化负对数似然。
1/G · Σ_i 对每个问题生成 G 个答案，计算 G 个损失的平均值。
min(r_i(θ)·A_i, clip(...)·A_i) 这是 PPO 的核心思想：clip 防止策略更新太激进。A_i 是 advantage（优势），衡量这个答案比组内平均好多少。
A_i = r_i − mean(r)/std(r) 这是 GRPO 的关键创新！不与旧策略比较，而是在组内标准化——把每个答案的奖励减去组平均值再除以标准差。这样就不需要单独的 Critic 网络，效率大幅提升！

1
冷启动（Cold Start）

用长 CoT 数据微调基础模型，解决 R1-Zero 可读性差的问题。数据来源：可读性过滤后的 R1-Zero 数据 + 人工后处理。

2
RL 阶段（面向推理）

GRPO + 奖励信号：
• 准确性奖励：答案正确得 +1，错误得 -1
• 格式奖励：模型必须把思考过程放在 <think>...</think> 标签内

3
SFT 阶段（面向通用能力）

用阶段 2 产生的数据 + 来自 DeepSeek-V3 的数据，训练模型在非推理任务（写作、对话、摘要）上的能力。

4
RL 阶段（面向全部能力）

在阶段 3 基础上，用更通用的奖励信号做 RL：
• 有用性奖励：回答是否对用户有帮助
• 无害性奖励：回答是否安全

奖励信号设计

✓
准确性奖励（Outcome Reward）

二元判断：答案对错。最简单也最有效的奖励。模型学会在产生错误后"反思"并纠正。

🎯
过程奖励（Process Reward）

给推理的每一步打分，而非只看最终答案。帮助模型理解"哪一步走错了"。但实现难度大（需要过程标注数据）。

📐
格式奖励（Format Reward）

强制模型把思考过程放在特定标签内。确保输出可解析，也间接鼓励模型写出更完整的思考链。

模型架构

DeepSeek-V3-Base (671B) Dense Mixture-of-Experts 架构，37B 参数每次前向传播激活
MLA (Multi-head Latent Attention) DeepSeek 自研注意力机制，大幅降低 KV cache 内存
DeepSeekMoE 细粒度专家分割 + 共享专家隔离，在保持性能的同时降低激活参数
Distillation: 7B / 8B / 32B / 70B 蒸馏到小模型，首次证明 7B 模型也能涌现推理能力（AIME 52.5%）

理解检查

Q: GRPO 相比 PPO 的关键改进是什么？

A GRPO 使用更大的 batch size ✗ 错误。batch size 不是主要改进。
B GRPO 在组内做标准化（用组均值和标准差替代 Critic 网络），不需要单独的价值网络 ✓ 正确！这是 GRPO 的核心创新——省去了训练 Critic 的开销，效率大幅提升。
C GRPO 使用不同的强化学习算法框架 ✗ 错误。GRPO 仍是 on-policy PPO 类方法，只是对 advantage 估计做了创新。

反思（Reflection）

模型发现推理方向错误时，会主动回退并重新思考。在生成答案的过程中，可以看到"Wait, let me reconsider..."这样的自我纠正输出。

模型在给出最终答案前，会主动验证中间步骤的正确性。这大大减少了"粗心错误"。

局限与展望

Limitations & Future Work

DeepSeek-R1 的局限

论文坦诚地指出了当前方法的不足：

⚠️
语言一致性（Language Inconsistency）

R1 和 R1-Zero 在推理过程中经常出现语言混合（中英文混杂）。这在数学和代码场景问题不大，但在开放域对话中影响体验。

原因：训练数据以英文为主，但 RL 过程没有对语言一致性做惩罚。

⚠️
函数调用与格式输出

R1 在 function calling、JSON 输出、text editing 等任务上表现不如专门的微调模型。这是因为 RL 阶段主要聚焦推理任务。

⚠️
模型规模门槛

671B 参数的模型训练成本极高。虽然蒸馏到 7B 证明了可行性，但大模型的推理能力上限仍然是未知的。

⚠️
冷启动数据的质量依赖

DeepSeek-R1 需要依赖 R1-Zero 的可读性改善数据做冷启动。这个数据 pipeline 目前依赖人工后处理，难以完全自动化。

未来研究方向

👁️
多模态推理

将纯 RL 推理方法扩展到图像、视频、音频等多模态场景。能否让模型通过纯 RL 学会"看图推理"？

🎯
更精细的奖励信号

当前主要用二元准确性奖励。能否设计更细粒度的过程奖励（每一步都给分）？形式化验证器能在这里发挥关键作用。

🤝
多智能体 RL 推理

让多个推理模型协作：一个负责提出假设，一个负责验证，一个负责整合。能否通过多智能体 RL 进一步提升推理质量？

⚡
推理效率优化

DeepSeek-R1 生成的长思维链消耗大量推理算力。能否通过模型压缩或早停策略，在不损失性能的同时降低推理成本？

论文对整个领域的启示

CORE TAKEAWAY
"If the reward signal is correct,
supervision is not necessary for reasoning.
The model will figure out HOW to reason
by itself — through pure trial and error."
correct reward signal 不是教模型"什么是对的推理过程"，而是告诉模型"什么样的结果是好的"。模型自己会发现有效的推理策略。
pure trial and error RL 本质上是"试错"——模型生成多个答案，正确的被强化，错误的被惩罚。正是这个过程让推理能力涌现出来。
