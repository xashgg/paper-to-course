Method Deep-Dive

核心算法：GRPO

GRPO（Group Relative Preference Optimization）是 DeepSeek 团队自研的 RL 训练算法，是 R1 的心脏。

💬 GRPO 是如何工作的？
📋
问题

我有一道数学题 Q。我需要模型生成多个不同的答案。

🤖
Model (π_θ)

正在生成 G=8 个不同解：[S₁, S₂, ..., S₈]。大部分会错，但有些可能对！

✅
Reward Model

对每个解打分：[r₁=0, r₂=1, r₃=0, r₄=0, r₅=1, r₆=0, r₇=0, r₈=0]。两个正确！

⚖️
GRPO 损失函数

计算偏好：正确解得正权重，错误解得负权重。然后更新 π_θ，让下次生成更多正确答案。

GRPO 目标函数
L(θ) = − E_{(q,g)∼D} [ 1/G Σ_i min(r_i(θ)·A_i, clip(r_i(θ), 1−ε, 1+ε)·A_i) ]

A_i = r_i − mean([r_1,...,r_G]) / std([r_1,...,r_G])
−E_{(q,g)∼D} 对训练集中的每个问题 q 和采样组 g，求期望（平均）损失。
1/G · Σ_i 对每个问题生成 G 个答案，计算 G 个损失的平均值。
min(r_i(θ)·A_i, clip(...)·A_i) 这是 PPO 的核心思想：clip 防止策略更新太激进。A_i 是 advantage（优势），衡量这个答案比组内平均好多少。
A_i = r_i − mean(r)/std(r) 这是 GRPO 的关键创新！不与旧策略比较，而是在组内标准化——把每个答案的奖励减去组平均值再除以标准差。这样就不需要单独的 Critic 网络，效率大幅提升！

1
冷启动（Cold Start）

用长 CoT 数据微调基础模型，解决 R1-Zero 可读性差的问题。数据来源：可读性过滤后的 R1-Zero 数据 + 人工后处理。

2
RL 阶段（面向推理）

GRPO + 奖励信号：
• 准确性奖励：答案正确得 +1，错误得 -1
• 格式奖励：模型必须把思考过程放在 <think>...</think> 标签内

3
SFT 阶段（面向通用能力）

用阶段 2 产生的数据 + 来自 DeepSeek-V3 的数据，训练模型在非推理任务（写作、对话、摘要）上的能力。

4
RL 阶段（面向全部能力）

在阶段 3 基础上，用更通用的奖励信号做 RL：
• 有用性奖励：回答是否对用户有帮助
• 无害性奖励：回答是否安全

奖励信号设计

✓
准确性奖励（Outcome Reward）

二元判断：答案对错。最简单也最有效的奖励。模型学会在产生错误后"反思"并纠正。

🎯
过程奖励（Process Reward）

给推理的每一步打分，而非只看最终答案。帮助模型理解"哪一步走错了"。但实现难度大（需要过程标注数据）。

📐
格式奖励（Format Reward）

强制模型把思考过程放在特定标签内。确保输出可解析，也间接鼓励模型写出更完整的思考链。

模型架构

DeepSeek-V3-Base (671B) Dense Mixture-of-Experts 架构，37B 参数每次前向传播激活
MLA (Multi-head Latent Attention) DeepSeek 自研注意力机制，大幅降低 KV cache 内存
DeepSeekMoE 细粒度专家分割 + 共享专家隔离，在保持性能的同时降低激活参数
蒸馏: 7B / 8B / 32B / 70B 蒸馏到小模型，首次证明 7B 模型也能涌现推理能力（AIME 52.5%）

理解检查

Q: GRPO 相比 PPO 的关键改进是什么？

A GRPO 使用更大的 batch size ✗ 错误。batch size 不是主要改进。
B GRPO 在组内做标准化（用组均值和标准差替代 Critic 网络），不需要单独的价值网络 ✓ 正确！这是 GRPO 的核心创新——省去了训练 Critic 的开销，效率大幅提升。
C GRPO 使用完全不同的强化学习算法框架 ✗ 错误。GRPO 仍是 on-policy PPO 类方法，只是对 advantage 估计做了创新。
