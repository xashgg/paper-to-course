发展脉络

Literature Timeline

LLM 推理能力的演进经历了四个关键阶段，每一次都突破了上一步的瓶颈。让我们用时间线回顾这条路径。

2022

Chain-of-Thought Prompting

Wei et al. 发现：让模型"一步步思考"（CoT），比直接给答案效果好得多。这是推理增强的开端——但这只是 prompting 技巧，不能真正教会模型推理。

CoT Prompting
2022

RLHF / InstructGPT

OpenAI 通过人类反馈（RLHF）训练 InstructGPT，让模型学会遵循指令。但 RLHF 依赖大量人工标注的偏好数据，成本极高。

RLHF
2023

GPT-4 + Tool Use

GPT-4 展示了强大的推理能力，但本质上仍是 SFT + RLHF 的产物。Code Interpreter 等工具帮助模型"作弊"——用外部工具弥补推理不足。

SFT + RLHF
2024.09

OpenAI o1 — Extended Thinking

OpenAI 推出 o1，首次在推理时引入"思考 token"机制，在 AIME 数学竞赛上达到 74.6% 准确率。但模型内部如何实现推理仍是黑箱。

Extended Thinking
2024.08

AlphaProof (DeepMind)

DeepMind 用形式化验证器 + RL 在 IMO 数学竞赛题上达到银牌水平。验证器的准确性是瓶颈——但证明了 RL 在形式化数学领域的潜力。

Formal Verification + RL
2025.01

DeepSeek-R1-Zero — 纯 RL

本文关键突破：DeepSeek-R1-Zero 完全不用 SFT，纯 RL 训练，涌现出反思能力。这是领域内首次证明"推理可以不依赖标注数据涌现出来"。

Pure RL ✅
2025.01

DeepSeek-R1 — 冷启动 + 多阶段

DeepSeek-R1 增加冷启动阶段（高质量 SFT 数据），通过四阶段训练，在推理 benchmark 上与 o1 持平，并开源全系列模型（7B~671B）。

DeepSeek-R1

核心洞察
所有先前方法都需要标注推理数据（SFT）。
R1-Zero 证明：只要奖励信号正确，
推理能力就会无需监督地涌现。
正确的奖励信号 R1-Zero 使用的奖励信号非常简单：正确性（outcome reward）——模型必须产生正确答案才能获得奖励。这比教模型"怎么推理"要简单得多。
推理涌现 模型自发地学会了"反思"——当发现中间步骤不对时，会回退重新思考。这种行为从未被显式教过，是 RL 过程中涌现的。

理解检查

Q: 为什么 DeepSeek-R1-Zero 被认为是领域的关键突破？

A 它使用了比 OpenAI o1 更多的标注数据 ✗ 错误。R1-Zero 恰恰是完全不用标注数据的。
B 它首次证明推理能力可以通过纯 RL 从零涌现，不需要任何监督数据 ✓ 正确！这是从未有人做到过的。
C 它训练了比 o1 更大的模型 ✗ 错误。模型规模相近，但方法论完全不同。
